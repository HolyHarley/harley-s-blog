# ブログ執筆スキル

このスキルは、Astro プロジェクトで新しいブログ記事を効率的に作成・執筆するための手順とガイドラインを提供します。

## 概要

ブログ記事のテーマ決定から執筆、ファイル作成までの一連の流れをサポートします。

## 実行手順

1.  **テーマと構成の決定**
    *   記事のテーマ、ターゲット読者、主要キーワードを決定します。
    *   記事の構成案（見出しレベルでのアウトライン）を作成し、ユーザーの確認を得ます。

2.  **ファイル作成**
    *   `src/content/blog/` ディレクトリに新しい`.md`または`.mdx` ファイルを作成します。
    *   ファイル名は `YYYY-MM-DD-slug.md` の形式、または記事の内容を表す英数字の `slug.md` とします。

3.  **Frontmatter の設定**
    *   以下のテンプレートを使用して、記事のメタデータを設定します。

    ```yaml
    ---
    title: "記事のタイトル"
    description: "記事の概要・抜粋"
    pubDate: "YYYY-MM-DD" # 例: 2024-01-01
    updatedDate: ""       # 更新日があれば記入
    heroImage: "/blog-placeholder-1.jpg" # publicディレクトリ内の画像パス
    tags: ["タグ1", "タグ2"]
    ---
    ```

    *   `layout` プロパティは、プロジェクトの構成に合わせて適切に設定してください（例: `../../layouts/BlogPost.astro`）。

4.  **本文の執筆**
    *   Markdown (または MDX) 形式で本文を執筆します。
    *   見出しは `## (H2)` から開始し、階層構造を意識してください。
    *   コードブロックには適切な言語シンタックスハイライトを指定してください（例: \`\`\`typescript）。
    *   画像は `public/` フォルダに配置し、絶対パス（例: `/images/pic.jpg`）で参照します。

5.  **レビューと修正**
    *   誤字脱字、リンク切れ、画像の表示などを確認します。
    *   必要に応じて修正を行います。

## ルール

*   **ファイル配置**: 記事ファイルは必ず `src/content/blog/` 配下に配置すること。
*   **画像パス**: 画像は `public/` ディレクトリに保存し、記事内では `/` から始まる絶対パスで参照すること。
*   **Frontmatter**: 必須項目（title, pubDate, description）は必ず含めること。
